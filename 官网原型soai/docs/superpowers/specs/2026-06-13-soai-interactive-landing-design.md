# SOAI 交互式落地页设计文档

**日期**：2026-06-13  
**项目**：官网原型 soai  
**目标**：替代现有静态介绍页，打造以骑手为主角的沉浸式科技感交互网页

---

## 一、设计定位

**核心诉求**：不是小程序演示结构，而是给骑手带来视觉和形象的冲击力。  
**受众**：马术骑手——既是产品用户，也是被打动的对象。  
**情绪目标**：骑手打开页面后感受到「这个系统真的在看见我」。

---

## 二、视觉语言

### 色彩系统

| 角色 | 颜色 | 用途 |
|------|------|------|
| 主背景 | `#06000f`（近黑深紫） | 全页底色 |
| 数据蓝 | `#00c8ff` | 骨骼线、数据标签、第一章强调色 |
| 渐变紫 | `#a855f7` | 后半段叙事、CTA、潜力感 |
| 过渡蓝 | `#6496ff` | 蓝→紫渐变中间色 |
| 数据绿 | `#00ff88` | 正向数据指标（稳定性高、评分好） |
| 警示红 | `#ff6b6b` | 需改进的数据指标 |
| 白/浅灰 | `rgba(255,255,255,0.85)` | 主文案 |

渐变方向：蓝（上/左）→ 紫（下/右），贯穿全页，象征「数据科学 → 个人潜力」。

### 网格与光晕

- 全页极低透明度网格（`28px × 28px`），用 CSS `background-image` 实现，透明度 ≤ 6%
- 关键章节底部有放射状光晕（`radial-gradient`），颜色随章节变化（蓝→紫）
- 浮动粒子（Canvas），密度极低，鼠标移动轻微偏移，强化科技氛围，不抢内容

### 字体排版

- 标题：系统字体栈，超粗（`font-weight: 900`），大字号（`clamp(48px, 8vw, 96px)`）
- 数据标签：`7–9px`，`letter-spacing: 2–4px`，全大写
- 正文：`15–16px`，行高 `1.7`，`rgba(255,255,255,0.6)`

## 三、页面章节结构

### Section 01 · 全屏开场

**叙事**：「你踩上马镫的那一刻」

**布局**：
- 100vh 全屏，垂直居中
- 背景：`#06000f` + 网格 + 蓝色光晕（画面下方）
- 主标题分两行：第一行白色「你踩上马镫」，第二行蓝→紫渐变「的那一刻」
- 副文案（小字）：「SOAI 已经开始记录」
- 右侧/底部：使用 `assets/hero-equestrian-training.png`，叠加蓝色发光边框（`box-shadow` + `filter: brightness`），不裁剪，保持原始比例
- 底部：向下箭头 + 「向下探索」，轻微脉冲动画引导滚动

**动效**：
- 进场：标题逐字从下淡入（`opacity 0→1` + `translateY 20px→0`，stagger 0.08s）
- 粒子从加载即开始漂浮

---

### Section 02 · 滚动骨骼揭示

**叙事**：「SOAI 开始看见你」

**核心交互**：ScrollTrigger 锁帧（`pin: true`）——页面滚动时视口锁定在此章节，骨骼随滚动进度比例生长，滚完自动解锁继续。

**骨骼结构**（SVG 路径，马术骑坐姿势）：
- 骑手处于坐姿：上半身直立，髋部弯曲，腿部向下踩镫
- 头部圆圈
- 脊柱（竖线，略前倾 5–10°）→ 双肩（横线）→ 双臂（自然下垂持缰）→ 双手
- 髋部 → 大腿（向前水平延伸）→ 小腿（垂直向下）→ 脚踝（踩镫点）
- 生长顺序：脊柱（0–20%）→ 头部（20–30%）→ 肩臂（30–50%）→ 髋腿（50–80%）→ 脚踝（80–100%）
- 用 `stroke-dashoffset` 动画实现描边生长

**数据标签**：
- 每段骨骼生长完成时，对应数据标签从侧边淡入
- 标签格式：`● 背部角度 ── 12.4°`（蓝色）/ `● 髋部稳定 ── 87%`（绿色）/ `● 重心偏移 ── −2.8cm`（红色）

**节点悬停交互**：
- 骨骼关节（圆点）悬停时：放大 + 发光 + 弹出 tooltip 卡片
- Tooltip 内容：关节名称、当前角度/数值、AI 建议一句话
- 点击关节：tooltip 固定显示（不需要持续悬停）

**左侧文案**（随滚动进度切换）：
- 0–33%：「AI 正在扫描你的姿态」
- 33–66%：「每一个关节都在记录」  
- 66–100%：「你的骑乘数据已完整捕获」

---

### Section 03 · 进步对比

**叙事**：「三周后的你」

**布局**：
- 左侧：动态折线图（SVG），两条曲线——「训练前」（暗色平缓）vs「使用 SOAI 后」（蓝紫渐变上扬）
- 右侧：三个数字统计卡片，进入视口时滚动计数：
  - `87%` 姿态稳定性提升
  - `3` 周平均见效周期
  - `12°` 背部角度优化

**动效**：
- 折线图：进入视口时路径从左向右描边生长（`stroke-dashoffset`）
- 数字：`0 → 目标值` 的滚动计数，`easeOut` 缓动，1.5s 完成

### Section 04 · CTA 收尾

**叙事**：「成为更好的骑手」

**布局**：
- 100vh 全屏，深紫底色 + 强烈紫色光晕居中
- 主标题：「成为更好的骑手」（白色大字）
- 副文案：「预约演示，亲身体验 AI 姿态分析」
- 主按钮：发光描边按钮「预约演示 →」（蓝→紫渐变边框，hover 时填充）
- 次按钮：「了解更多」（透明描边）

## 四、交互规格

### 滚动系统

- 使用 GSAP ScrollTrigger（CDN 引入，无需构建工具）
- Section 02 使用 `scrub: true` + `pin: true`，实现滚动锁帧骨骼生长
- 其他章节使用 `once: true` 触发进场动画

### 骨骼动画

- SVG 路径预先计算 `stroke-dasharray` = 路径总长度（`getTotalLength()`）
- 初始 `stroke-dashoffset` = 路径总长度（不可见）
- ScrollTrigger `progress` 驱动 `stroke-dashoffset` 从全长 → 0

### 粒子系统

- Canvas 全屏覆盖（`pointer-events: none`）
- 粒子数量：50–80 个，半径 1–2px，颜色 `rgba(0,200,255,0.15)` 到 `rgba(168,85,247,0.1)`
- 每帧微移，鼠标位置影响最近粒子的漂移方向（半径 150px 内）

### 无障碍

- `@media (prefers-reduced-motion: reduce)`：关闭所有滚动触发动画，仅保留 `opacity` 淡入
- 骨骼 SVG 有 `aria-label`，数据标签有语义文本
- 导航键盘可访问，focus 环可见
