# SOAI API 阿里云/腾讯云部署清单

## 1. 服务器基础环境

建议先在腾讯云服务器安装：

```bash
node -v
npm -v
ffmpeg -version
pm2 -v
nginx -v
```

如果缺少：

```bash
sudo apt update
sudo apt install -y nodejs nginx ffmpeg git
sudo npm install -g pm2
```

要求 Node.js 版本不低于 16。

如果服务器是 Alibaba Cloud Linux / CentOS 7，系统 `glibc` 较旧，不建议强行安装 Node.js 20。当前服务已兼容 Node.js 16，可直接使用宝塔或系统已有的 Node.js 16。

## 2. 拉取代码

推荐目录：

```bash
mkdir -p /www/wwwroot
cd /www/wwwroot
git clone git@github.com:ouyangliteng/soai-frontend.git
cd /www/wwwroot/soai-frontend/后端Mock服务
npm test
```

## 3. 创建视频存储目录

```bash
sudo mkdir -p /data/soai-storage/uploads /data/soai-storage/frames /data/soai-storage/db
sudo chown -R $(whoami):$(whoami) /data/soai-storage
```

## 4. 启动后端 API

```bash
cd /www/wwwroot/soai-frontend/后端Mock服务
npm install
pm2 start deploy/ecosystem.config.cjs
pm2 save
pm2 status
```

如果要启用真实 YOLO-Pose 服务，先启动 Python 服务：

```bash
cd /www/wwwroot/soai-frontend/PythonPose服务
docker compose -f docker-compose.aliyun.yml up -d --build
curl http://127.0.0.1:8793/health
```

本机检查：

```bash
curl http://127.0.0.1:8787/health
curl http://127.0.0.1:8787/api/student/profile
```

后续每次从 GitHub 更新服务器，可以直接执行：

```bash
cd /www/wwwroot/soai-frontend/后端Mock服务
bash deploy/update-server.sh
```

## 5. 配置 Nginx

将 `deploy/nginx-api.soai.yun.conf` 放到 Nginx 站点配置中，或在宝塔面板里按同等内容配置：

关键配置：

- `client_max_body_size 180m`
- `proxy_pass http://127.0.0.1:8787`
- `X-Forwarded-Proto https`
- `/storage/` 指向 `/data/soai-storage/`

重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

外网检查：

```bash
curl https://api.soai.yun/health
curl https://api.soai.yun/api/student/profile
```

或者执行完整验收脚本，第二个参数是后台 token：

```bash
cd /www/wwwroot/soai-frontend/后端Mock服务
bash deploy/verify-api.sh https://api.soai.yun "$SOAI_ADMIN_TOKEN"
```

## 6. 微信小程序后台域名

微信公众平台需要配置：

- request 合法域名：`https://api.soai.yun`
- uploadFile 合法域名：`https://api.soai.yun`
- downloadFile 合法域名：`https://api.soai.yun`

## 7. 小程序 API 模式

当前代码默认 API 地址已经是：

```text
https://api.soai.yun
```

如果体验版曾经写入过旧地址，可在开发者工具 Console 清理：

```js
wx.setStorageSync("soai_data_mode", "api")
wx.setStorageSync("soai_api_base_url", "https://api.soai.yun")
```

## 8. 验收主链路

1. 学员登录并补全资料。
2. 上传 10-60 秒、150MB 以内视频。
3. 后端生成上传记录和分析任务。
4. 查询任务进入抽帧、姿态识别、规则引擎、报告生成阶段。
5. 学员看到报告。
6. 教练复核并提交 AI 判断准确性标注。
7. 后台可查询标注数据：

```bash
curl https://api.soai.yun/api/coach/review-annotations
```

## 9. 重要说明

当前部署版已经具备真实上传、抽帧、Python 姿态服务接口、YOLO-Pose、规则引擎和标注链路。

姿态识别建议生产环境使用：

```text
SOAI_POSE_PROVIDER=http
SOAI_POSE_SERVICE_URL=http://127.0.0.1:8793
SOAI_POSE_MODEL_PROVIDER=yolo-pose
```

如果要更稳妥地做第一轮服务器冒烟测试，可以临时使用：

```text
SOAI_POSE_PROVIDER=auto
SOAI_POSE_MODEL_PROVIDER=synthetic
```

这样 Node 会优先调用独立 Python Pose Service，服务不可用时回退到本地模拟关键点。

## 10. 阿里云 OSS 直传

第一轮建议先用本地存储跑通服务器链路：

```text
SOAI_STORAGE_PROVIDER=local
SOAI_STORAGE_ROOT=/data/soai-storage
SOAI_STORAGE_PUBLIC_BASE_URL=https://api.soai.yun/storage
```

链路稳定后切换 OSS：

```text
SOAI_STORAGE_PROVIDER=aliyun-oss
ALIYUN_OSS_BUCKET=你的 bucket
ALIYUN_OSS_REGION=oss-cn-hangzhou
ALIYUN_OSS_ENDPOINT=https://你的 bucket.oss-cn-hangzhou.aliyuncs.com
ALIYUN_OSS_PUBLIC_BASE_URL=https://你的 bucket.oss-cn-hangzhou.aliyuncs.com
ALIYUN_ACCESS_KEY_ID=RAM 子账号 AccessKeyId
ALIYUN_ACCESS_KEY_SECRET=RAM 子账号 AccessKeySecret
```

后端会在 `/api/videos/upload-token` 返回 OSS POST Policy 和 `uploadFormData`，小程序用 `wx.uploadFile` 直传到 OSS。
