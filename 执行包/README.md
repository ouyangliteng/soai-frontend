## 推荐执行顺序

### 第一步：研发确认范围

使用文件：

- `V0.2内测可用版PRD.md`
- `开发任务拆解.md`

目标：

- 确认 V0.2 只做学员端可用化和轻量教练复核。
- 暂不做完整教练端、支付和复杂运营功能。

### 第二步：后端和 AI 对齐结构

使用文件：

- `数据结构与接口草案.md`
- `AI分析报告提示词与输出规范.md`
- `隐私与视频使用说明.md`
- `../后端Mock服务`

目标：

- 确认报告字段。
- 确认分析任务状态流。
- 确认 AI 输出必须是结构化 JSON。
- 使用 Mock API 先完成小程序联调。
- 确认视频用途授权和匿名案例授权的字段、展示和后端校验一致。

### 第三步：小程序开发

使用文件：

- `开发任务拆解.md`

目标：

- 第 1 周完成资料和上传稳定性。
- 第 2 周完成报告升级。
- 第 3 周完成趋势、埋点和内测准备。

### 第四步：内测验收

使用文件：

- `V0.2内测验收清单.md`

目标：

- 用真实学员视频测试。
- 用教练复核判断 AI 报告是否有价值。
- 达到 Go 条件后进入小范围内测。

## 本阶段最重要的产品判断

V0.2 不应把 AI 包装成“替代教练”，而是要做成专业、准确、可复核的训练辅助工具。

最重要的体验不是“分数高不高”，而是：

- 学员知道自己下次怎么练。
- 教练觉得报告有参考价值。
- 训练趋势能看出持续进步。
- 上传和分析过程稳定可信。
