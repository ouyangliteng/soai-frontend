# SOAI V0.2 内测日报模板

## 1. 使用场景

每天内测结束前，由运营负责人根据运营看板和 `GET /api/operations/daily-report` 输出一份日报。

日报目的不是写长报告，而是快速判断：

- 主链路今天是否稳定。
- 教练是否完成复核。
- 学员是否理解报告并愿意继续上传。
- 是否存在隐私、授权或 AI 表达风险。
- 明天最应该处理哪 3 件事。

## 2. 接口

```text
GET /api/operations/daily-report
```

返回字段：

| 字段 | 说明 |
| --- | --- |
| `date` | 日报日期 |
| `phase` | 当前阶段 |
| `conclusion` | 今日结论 |
| `goStatus` | `go` 或 `conditional_go` |
| `metrics` | 今日核心指标 |
| `reviewQuestions` | 每日 6 个复盘问题 |
| `topFeedbackTags` | 高频反馈标签 |
| `risks` | 风险项 |
| `nextActions` | 次日动作 |

## 4. Go / Conditional Go 解释

### `go`

表示主链路表现稳定，可以继续按当前内测节奏推进。

典型条件：

- 上传成功率达标。
- AI 分析成功率达标。
- 教练复核率达标。
- 反馈未出现明显隐私或报告质量风险。

### `conditional_go`

表示可以继续小范围内测，但必须跟进风险或短板。

常见原因：

- 教练复核率不足。
- 二次上传人数不足。
- 上传或分析指标低于目标。
- 出现视频授权、AI 表达或公开案例风险。

## 5. 明日动作优先级

如果日报中出现多个动作，按以下顺序处理：

1. 隐私、授权和公开表达风险。
2. 上传失败、分析失败、报告打不开等主链路问题。
3. 教练待复核报告。
4. 学员反馈中“看不懂”“建议不具体”等报告体验问题。
5. 二次上传提醒和内容发布。

## 6. 运营使用建议

- 每天固定时间输出，建议 22:00 前完成。
- 只记录事实和下一步，不写空泛总结。
- 每天最多保留 3 个明日动作，避免团队分散。
- 发现 AI 报告明显错误时，先暂停该案例对外传播。
- 发现视频授权疑虑时，先停止公开使用相关素材。
