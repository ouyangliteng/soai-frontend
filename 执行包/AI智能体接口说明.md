# AI 智能体接口说明

## 1. 当前目标

当前已经在 `../后端Mock服务/src/ai-agents.js` 中建立第一版 AI 智能体服务骨架。

这一版不调用外部大模型，而是使用确定性规则生成结构化输出，目标是先把产品链路和接口边界跑通：

- AI 训练报告草稿
- AI 报告质量校验
- 教练复核草稿
- 学员报告解读
- 运营内容草稿

后续接入真实大模型时，只需要替换 `ai-agents.js` 内部实现，接口可以保持稳定。

## 2. 接口列表

### 2.1 生成 AI 报告草稿

`POST /api/ai/report-draft`

用途：

- 根据学员资料、视频分析结果、历史报告生成结构化训练报告。
- 当前可传入完整输入，也可传空对象使用 Mock 数据。

响应：

```json
{
  "report": {},
  "validation": {
    "valid": true,
    "issues": []
  }
}
```

### 2.2 校验 AI 报告

`POST /api/ai/report-validate`

用途：

- 检查 AI 输出是否符合产品要求。
- 防止报告缺失关键字段或出现禁用表达。

检查项：

- 总体评分
- 问题点
- 风险点
- 本次进步
- 3 到 5 条下次训练重点
- 局限性说明
- 禁用表达

### 2.3 生成教练复核草稿

`GET /api/ai/reports/{reportId}/coach-review-draft`

用途：

- 给教练提供复核意见草稿。
- 提炼下次训练重点。
- 提醒是否必须由教练确认。

原则：

- 草稿不能自动发送给学员。
- 教练必须确认后才进入正式报告。

### 2.4 生成学员报告解读

`GET /api/ai/reports/{reportId}/student-explanation`

用途：

- 把专业报告转成学员更容易理解的解释。
- 帮助学员知道下次训练先关注什么。

边界：

- 不替代教练。
- 风险点仍提示结合教练判断。

### 2.5 生成运营内容草稿

`GET /api/ai/reports/{reportId}/operation-content`

用途：

- 从匿名报告中生成小红书、抖音、微信群内容草稿。
- 帮助运营快速整理真实训练案例。

边界：

- 不公开学员身份。
- 不夸大 AI 准确性。
- 案例发布前必须获得授权。

## 3. 后续接入真实大模型

建议顺序：

1. 保留接口不变。
2. 在 `ai-agents.js` 内增加模型调用层。
3. 调用模型后必须继续执行 `validateTrainingReport`。
4. 校验失败时进入重试或人工复核。
5. 所有公开内容必须再经过运营人工确认。

## 4. 安全边界

AI 智能体统一边界：

- 不说“完全安全”。
- 不说“无需教练”。
- 不做医学诊断。
- 不承诺训练效果。
- 不公开个人隐私。
- 不把风险提醒写成恐吓表达。
