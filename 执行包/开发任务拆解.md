# V0.2 开发任务拆解

## 1. 第 1 周：资料与上传稳定

### 1.1 学员资料

- 建立 `StudentProfile` 数据模型。
- 新建学员资料页。
- 新建资料编辑能力。
- 增加首次上传前资料校验。
- 报告生成时保存 `studentSnapshot`。

验收：

- 新用户能填写资料。
- 老用户能编辑资料。
- 未填必填项不能上传。

### 1.2 视频上传

- 接入视频选择。
- 增加格式校验：mp4、mov。
- 增加大小校验：建议 300 MB 上限。
- 增加时长校验：10 秒到 60 秒。
- 增加上传进度。
- 增加上传失败重试。
- 上传成功后创建 `TrainingVideo`。

验收：

- 上传过程状态清晰。
- 失败可重试。
- 视频过大时上传前拦截。

### 1.3 分析任务状态

- 建立 `AnalysisTask` 数据模型。
- 上传成功后创建分析任务。
- 轮询或订阅任务状态。
- 支持分析中状态恢复。
- 支持分析失败重试。

验收：

- 分析中退出再进入能恢复。
- 分析失败可重新分析。

## 2. 第 2 周：报告升级

### 2.1 报告结构

- 建立 `TrainingReport` 数据模型。
- 接入 AI 输出 JSON 规范。
- 报告存储结构化字段。
- 增加评分维度。

验收：

- 报告可展示总体评分、问题点、风险点、进步、训练重点。

### 2.2 报告页面

- 顶部展示学员信息和训练时间。
- 展示总体评分和一句话结论。
- 展示问题点列表。
- 展示风险点列表。
- 展示本次进步。
- 展示下次训练重点。
- 展示教练复核模块。

验收：

- 报告第一屏能看到核心结论。
- 风险点和问题点视觉上区分。

### 2.3 教练复核轻量能力

- 建立 `CoachReview` 数据模型。
- 报告页显示复核状态。
- 支持提交教练意见。
- 保留 AI 原始报告，不被教练意见覆盖。

验收：

- 教练意见提交后可展示。
- 可追踪教练名称和复核时间。

## 3. 第 3 周：趋势与内测

### 3.1 历史趋势

- 汇总历史报告分数。
- 展示最近 5 次趋势。
- 训练达到 10 次后展示最近 10 次趋势。
- 生成趋势总结。
- 少于 2 次时展示空状态。

验收：

- 趋势数据与历史报告一致。
- 趋势结论不是固定文案。

### 3.2 埋点与监控

- 增加上传相关埋点。
- 增加分析相关埋点。
- 增加报告阅读埋点。
- 增加复核提交埋点。
- 输出每日内测数据表。

验收：

- 能统计上传成功率、分析成功率、报告打开率、二次上传率。

### 3.3 内测准备

- 准备内测说明。
- 准备隐私说明。
- 准备反馈表。
- 确定内测学员和教练名单。
- 建立问题记录表。

验收：

- 至少 10 名学员可参与内测。
- 至少 2 名教练参与复核。

## 4. 优先级总表

| 优先级 | 模块 | 任务 | 是否阻塞内测 |
| --- | --- | --- | --- |
| P0 | 学员资料 | 资料填写、编辑、必填校验 | 是 |
| P0 | 视频上传 | 上传校验、进度、重试 | 是 |
| P0 | AI 分析 | 任务创建、状态恢复、失败重试 | 是 |
| P0 | 报告 | 问题点、风险点、进步、训练重点 | 是 |
| P0 | 报告 | 教练复核意见展示 | 是 |
| P0 | 趋势 | 最近 5 次趋势 | 是 |
| P1 | 趋势 | 最近 10 次趋势 | 否 |
| P1 | 运营 | 反馈表和每日数据表 | 否 |
| P2 | 分享 | 报告导出图片 | 否 |
