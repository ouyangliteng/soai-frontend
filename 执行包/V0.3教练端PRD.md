# 马术 AI 教学小程序 V0.3 教练端 PRD

## 1. 版本定位

V0.3 的目标是把 V0.2 的“学员上传和 AI 报告”升级为“教练可管理、可复核、可跟进”的训练协作系统。

核心链路：

学员上传训练视频 -> AI 生成报告 -> 教练查看待复核报告 -> 教练复核 -> 布置下次训练重点 -> 学员继续上传 -> 趋势追踪

V0.3 不做复杂俱乐部管理和排课系统，先做教练端最小闭环。

## 2. 用户角色

### 2.1 教练

核心需求：

- 快速看到哪些学员有新报告。
- 优先处理高风险或低评分报告。
- 查看单个学员最近训练趋势。
- 对 AI 报告进行复核。
- 给学员布置下次训练重点。

### 2.2 学员

核心需求：

- 看到教练复核后的最终建议。
- 知道下次训练重点。
- 了解自己最近是否进步。

### 2.3 运营人员

核心需求：

- 看到教练是否参与复核。
- 判断 AI 报告是否真的被教练使用。
- 为内容推广提炼真实案例。

## 3. V0.3 成功标准

- 教练可以看到学员列表。
- 教练可以看到待复核报告列表。
- 教练可以按风险优先级处理报告。
- 教练可以进入学员详情，查看最近 5 次训练趋势。
- 教练可以提交复核意见和下次训练重点。
- 学员报告页可以显示教练复核意见。
- 运营可以统计教练复核率。

建议指标：

- 教练复核率 >= 60%
- 高风险报告 24 小时内复核率 >= 80%
- 教练平均复核耗时 <= 3 分钟/份
- 学员查看教练意见比例 >= 70%

## 4. 功能范围

### 4.1 P0 必做

- 教练工作台
- 待复核报告列表
- 学员列表
- 学员详情
- 学员最近 5 次趋势
- 报告详情复核
- 下次训练重点编辑
- 复核状态同步到学员端

### 4.2 P1 建议

- 风险优先级排序
- 按学员等级筛选
- 按复核状态筛选
- 快捷复核模板
- 学员训练任务完成状态

### 4.3 暂不做

## 5. 页面结构

### 5.1 教练工作台

核心模块：

- 今日待复核数量
- 高风险报告数量
- 今日已复核数量
- 学员二次上传人数
- 待处理报告列表
- 学员列表入口

排序建议：

1. 高风险报告
2. 低评分报告
3. 新提交报告
4. 长时间未复核报告

### 5.2 待复核列表

字段：

- 学员姓名
- 当前等级
- 训练日期
- 总评分
- 风险等级
- AI 一句话结论
- 距提交时间
- 复核状态

操作：

- 查看报告
- 快速标记无需复核
- 进入学员详情

### 5.3 学员列表

字段：

- 学员姓名
- 当前等级
- 最近训练时间
- 最近评分
- 趋势状态
- 待复核数量

筛选：

- 全部学员
- 有待复核
- 最近 7 天活跃
- 评分下降

### 5.4 学员详情

模块：

- 学员基础资料
- 最近一次报告摘要
- 最近 5 次趋势
- 主要反复问题
- 风险点出现频次
- 训练记录列表
- 教练给出的训练重点

### 5.5 教练复核页

模块：

- AI 报告摘要
- 视频信息
- 问题点
- 风险点
- AI 下次训练重点
- 教练复核意见
- 教练确认的训练重点
- 复核提交按钮

复核状态：

- 待复核
- 已复核
- 无需复核

## 6. 复核规则

必须复核：

- AI 标记中高风险。
- 总评分低于 70。
- 学员连续两次评分下降。
- AI 置信度为 low。

建议复核：

- 新学员前三次训练。
- 学员上传了新的训练动作类型。
- 学员主动申请教练点评。

可无需复核：

- 低风险、稳定提升、教练已在现场点评过。

## 8. 教练端接口草案

### 8.1 获取教练工作台

`GET /api/coach/dashboard`

响应：

```json
{
  "stats": {},
  "pendingReports": [],
  "activeStudents": []
}
```

### 8.2 获取待复核报告

`GET /api/coach/reports?status=pending`

响应：

```json
{
  "items": []
}
```

### 8.3 获取教练学员列表

`GET /api/coach/students`

响应：

```json
{
  "items": []
}
```

### 8.4 获取学员训练详情

`GET /api/coach/students/{studentId}`

响应：

```json
{
  "profile": {},
  "latestReport": {},
  "trend": {},
  "reports": []
}
```

### 8.5 提交复核和训练重点

`POST /api/coach/reports/{reportId}/review`

请求：

```json
{
  "status": "reviewed",
  "comment": "AI 对上身稳定性的判断基本准确，下次加强轻快步节奏。",
  "focusItems": [
    "轻快步节奏",
    "小腿稳定"
  ],
  "assignmentComment": "下次训练前 10 分钟做轻快步节奏稳定练习。"
}
```

## 9. 验收标准

- 教练能在 3 次点击内打开待复核报告。
- 教练能在复核页看到 AI 问题点、风险点和训练重点。
- 教练提交复核后，学员端报告同步显示。
- 学员详情能展示最近 5 次趋势。
- 高风险报告有明显标识。
- 教练意见不会覆盖 AI 原始报告。
- 复核记录包含教练、时间、状态和意见。
