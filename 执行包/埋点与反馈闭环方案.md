# SOAI 小程序埋点与反馈闭环方案

## 1. 目标

把“学员端上传视频 -> AI 分析 -> 展示报告 -> 教练复核 -> 二次上传”变成可验证、可复盘、可运营的主链路。

本方案服务三个判断：

1. 产品是否可用：上传、分析、报告展示是否稳定。
2. 教学是否有价值：教练是否认可报告，学员是否知道下次怎么练。
3. 运营是否能转化：内容、微信群、小程序、天猫咨询之间是否形成闭环。

## 2. 事件字典

### 2.1 学员资料

| 事件 | 触发时机 | 必带字段 | 用途 |
| --- | --- | --- | --- |
| `profile_view` | 进入资料页 | `userId`, `studentId`, `page` | 判断资料补全入口是否被看到 |
| `profile_save_success` | 资料保存成功 | `studentId`, `properties.hasCoach`, `properties.hasClub` | 计算资料保存成功率和资料完整度 |

### 2.2 视频上传

| 事件 | 触发时机 | 必带字段 | 用途 |
| --- | --- | --- | --- |
| `video_select` | 选择视频成功 | `studentId`, `properties.sizeMb`, `properties.durationSec`, `properties.format` | 判断用户是否开始上传 |
| `video_validate_failed` | 格式、大小、时长校验失败 | `properties.reason` | 发现视频规格和提示问题 |
| `video_upload_start` | 点击开始上传 | `studentId` | 计算上传漏斗起点 |
| `video_upload_success` | 上传完成并创建视频记录 | `studentId`, `videoId`, `taskId` | 计算上传成功率 |
| `video_upload_failed` | 上传失败或创建任务失败 | `properties.message` | 定位上传失败原因 |

### 2.3 AI 分析

| 事件 | 触发时机 | 必带字段 | 用途 |
| --- | --- | --- | --- |
| `analysis_task_created` | 分析任务创建成功 | `taskId`, `videoId` | 计算进入 AI 分析的人数 |
| `analysis_success` | 分析完成并生成报告 | `taskId`, `reportId` | 计算分析成功率 |
| `analysis_failed` | 分析失败或超时 | `taskId`, `properties.message` | 定位 AI 服务稳定性问题 |
| `retry_click` | 用户点击重试 | `taskId`, `properties.target` | 判断失败恢复是否有效 |

### 2.4 报告与趋势

| 事件 | 触发时机 | 必带字段 | 用途 |
| --- | --- | --- | --- |
| `report_view` | 进入报告页 | `reportId`, `studentId` | 计算报告打开率 |
| `report_read_complete` | 报告停留达到 30 秒或滑到底部 | `reportId`, `properties.durationSec` | 判断报告是否被认真阅读 |
| `trend_view` | 进入趋势页 | `studentId`, `properties.limit`, `properties.recordCount` | 判断趋势功能是否有使用价值 |

### 2.5 教练复核与反馈

| 事件 | 触发时机 | 必带字段 | 用途 |
| --- | --- | --- | --- |
| `coach_review_submit` | 教练提交复核意见 | `coachId`, `reportId`, `properties.focusCount` | 计算教练复核率 |
| `feedback_submit` | 学员或教练提交反馈 | `role`, `reportId`, `properties.rating` | 形成问题池和质量评估 |

## 3. 统一字段

每个事件统一使用以下字段：

```json
{
  "eventId": "event_001",
  "eventName": "report_view",
  "userId": "wx_user_001",
  "studentId": "student_001",
  "coachId": "coach_001",
  "page": "report",
  "videoId": "video_001",
  "taskId": "task_001",
  "reportId": "report_001",
  "channel": "wechat_group",
  "properties": {},
  "occurredAt": "2026-06-01T20:30:00+08:00"
}
```

要求：

- `eventName` 必须来自事件字典。
- `occurredAt` 使用 ISO 时间。
- 不在埋点中记录完整视频地址、手机号、身份证等敏感信息。
- 渠道来源建议枚举：`xiaohongshu`、`douyin`、`wechat_video`、`wechat_group`、`tmall`、`website`、`offline_club`。

## 4. 指标口径

### 4.1 主链路指标

| 指标 | 计算方式 | V0.2 目标 |
| --- | --- | --- |
| 资料保存成功率 | `profile_save_success / profile_view` | >= 95% |
| 上传成功率 | `video_upload_success / video_upload_start` | >= 90% |
| AI 分析成功率 | `analysis_success / analysis_task_created` | >= 85% |
| 报告打开率 | `report_view / analysis_success` | >= 95% |
| 报告深度阅读率 | `report_read_complete / report_view` | >= 60% |
| 教练复核率 | `coach_review_submit / report_view` | >= 60% |
| 二次上传率 | 完成 2 次及以上 `video_upload_success` 的学员 / 上传成功学员 | >= 30% |

### 4.2 质量指标

| 指标 | 来源 | V0.2 目标 |
| --- | --- | --- |
| 教练认为有参考价值比例 | 教练反馈 `usefulnessRating >= 4` | >= 70% |
| 明显错误或编造比例 | 教练反馈标签 `明显错误`、`疑似编造` | <= 10% |
| 学员理解度 | 学员反馈 `rating >= 4` | >= 70% |
| 上传体验满意度 | 学员反馈 `uploadExperienceRating >= 4` | >= 80% |

### 4.3 运营转化指标

| 指标 | 计算方式 | 观察目的 |
| --- | --- | --- |
| 内容到微信群转化 | 微信群新增人数 / 内容曝光人数 | 判断内容是否吸引目标用户 |
| 微信群到内测申请 | 内测申请人数 / 微信群新增人数 | 判断私域承接有效性 |
| 内测到上传 | `video_upload_success` 学员数 / 内测申请人数 | 判断产品入口是否清楚 |
| 报告到产品咨询 | 产品咨询人数 / `report_view` 学员数 | 判断 AI 服务是否带动硬件和服务兴趣 |

## 5. 反馈表字段

### 5.1 学员反馈

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `role` | string | 固定为 `student` |
| `reportId` | string | 对应报告 |
| `rating` | number | 总体满意度，1 到 5 |
| `usefulnessRating` | number | 是否知道下次怎么练，1 到 5 |
| `uploadExperienceRating` | number | 上传体验，1 到 5 |
| `comment` | string | 自由反馈 |
| `tags` | string[] | 可选：上传慢、报告看不懂、建议具体、想继续上传、想咨询装备 |

### 5.2 教练反馈

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `role` | string | 固定为 `coach` |
| `reportId` | string | 对应报告 |
| `rating` | number | 总体评价，1 到 5 |
| `accuracyRating` | number | 动作问题准确性，1 到 5 |
| `usefulnessRating` | number | 教学参考价值，1 到 5 |
| `comment` | string | 复核补充 |
| `tags` | string[] | 可选：问题准确、建议可执行、风险表达谨慎、明显错误、视频角度不足 |

## 6. Mock API

当前 Mock API 已预留以下接口：

### 6.1 上报埋点

`POST /api/analytics/events`

请求示例：

```json
{
  "eventName": "report_view",
  "studentId": "student_001",
  "reportId": "report_001",
  "page": "report",
  "properties": {
    "entry": "analysis_complete"
  }
}
```

### 6.2 查询埋点

`GET /api/analytics/events?eventName=report_view`

### 6.3 提交反馈

`POST /api/feedback`

请求示例：

```json
{
  "role": "coach",
  "reportId": "report_001",
  "rating": 5,
  "accuracyRating": 4,
  "usefulnessRating": 5,
  "comment": "问题点有参考价值，但建议补充马匹状态信息。",
  "tags": ["问题准确", "视频角度不足"]
}
```

### 6.4 反馈汇总

`GET /api/feedback/summary`

## 7. 每日内测复盘表

建议每天 22:00 前由运营负责人汇总。

| 模块 | 每日填写项 |
| --- | --- |
| 主链路 | 新增报名、视频上传成功数、上传失败数、AI 成功数、AI 失败数、报告打开数 |
| 教练端 | 待复核数、已复核数、平均复核时长、教练认为有价值报告数 |
| 反馈 | 学员反馈数、教练反馈数、最高频问题、是否出现隐私或表达风险 |
| 运营 | 小红书/抖音/视频号曝光、微信群新增、内测申请、天猫咨询 |
| 次日动作 | 需要修复的问题、需要跟进的用户、需要发布的内容 |

## 8. 每周决策规则

### 8.1 继续扩大内测

满足以下条件可以从 10 到 20 人扩大到 50 人：

- 上传成功率 >= 90%。
- AI 分析成功率 >= 85%。
- 教练认为报告有参考价值比例 >= 70%。
- 没有严重隐私投诉。
- 至少 30% 学员完成第二次上传。

### 8.2 暂停推广

出现以下情况应暂停公开内容推广，先修复产品：

- 上传或分析失败集中出现。
- AI 报告出现明显编造或过度判断。
- 用户对视频授权或隐私用途产生明确疑虑。
- 教练普遍认为报告不可用。

## 9. 产品边界

- 埋点只服务产品质量、教学价值和运营复盘，不用于过度追踪个人行为。
- 公开案例必须单独取得授权。
- 任何报告相关内容都要保留“AI 辅助、教练复核”的表达边界。
- 指标好看不代表产品成功，最终判断仍以真实训练价值和教练认可为准。
