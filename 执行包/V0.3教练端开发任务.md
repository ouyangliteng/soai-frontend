# V0.3 教练端开发任务

## 1. 第 1 周：教练工作台

### 1.1 教练身份和入口

- 增加教练端入口。
- 建立教练 mock 数据或登录后的教练信息。
- 区分学员端和教练端页面。

验收：

- 教练可以进入工作台。
- 工作台显示待复核、已复核、高风险、活跃学员数量。

### 1.2 待复核报告列表

- 汇总 `coachReviewStatus = pending` 的报告。
- 展示学员、日期、评分、风险等级、一句话结论。
- 高风险和低评分优先排序。
- 支持点击进入报告复核。

验收：

- 待复核报告数量准确。
- 高风险报告标识明显。
- 点击后进入正确报告。

## 2. 第 2 周：学员管理

### 2.1 学员列表

- 展示教练名下学员。
- 展示最近训练日期、最近评分、趋势状态、待复核数量。
- 支持全部、有待复核、最近活跃、评分下降筛选。

验收：

- 学员列表数据与报告数据一致。
- 筛选结果准确。

### 2.2 学员详情

- 展示学员基础资料。
- 展示最近一次报告摘要。
- 展示最近 5 次趋势。
- 展示训练记录列表。
- 展示反复出现的问题点和风险点。

验收：

- 教练可以快速判断学员是否进步。
- 学员详情能进入具体报告。

## 3. 第 3 周：复核和训练任务

### 3.1 复核提交

- 复核页展示 AI 报告摘要。
- 教练填写复核意见。
- 教练确认下次训练重点。
- 提交后更新报告复核状态。

验收：

- 教练意见保存在独立字段。
- 学员端报告同步展示教练意见。

### 3.2 训练重点布置

- 建立 `TrainingAssignment` 数据结构。
- 支持从复核页生成下次训练重点。
- 学员端报告页显示教练确认后的训练重点。

验收：

- 训练重点可追踪。
- 不覆盖 AI 原始建议。

## 4. 优先级

| 优先级 | 模块 | 任务 | 是否阻塞 V0.3 |
| --- | --- | --- | --- |
| P0 | 工作台 | 统计和待复核入口 | 是 |
| P0 | 报告 | 待复核列表 | 是 |
| P0 | 复核 | 教练意见提交 | 是 |
| P0 | 学员 | 学员详情和趋势 | 是 |
| P1 | 筛选 | 复核状态和活跃筛选 | 否 |
| P1 | 任务 | 下次训练重点布置 | 否 |
| P2 | 运营 | 教练复核排行榜 | 否 |
