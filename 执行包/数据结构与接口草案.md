# 马术 AI 教学小程序数据结构与接口草案

## 1. 设计原则

- 历史报告必须保留当时的学员资料快照，避免资料修改后影响旧报告。
- 上传任务、分析任务、报告结果要分开存储，便于失败重试和状态恢复。
- AI 报告输出必须结构化，方便小程序展示和后续趋势计算。
- 教练复核意见独立存储，便于后续教练端扩展。

## 2. 数据模型

### 2.1 StudentProfile

学员当前资料。

```json
{
  "id": "student_001",
  "userId": "wx_user_001",
  "name": "张同学",
  "age": 14,
  "heightCm": 160,
  "weightKg": 48,
  "ridingYears": 2,
  "currentLevel": "初级进阶",
  "coachId": "coach_001",
  "coachName": "李教练",
  "clubId": "club_001",
  "clubName": "示例马术俱乐部",
  "createdAt": "2026-06-01T10:00:00+08:00",
  "updatedAt": "2026-06-01T10:00:00+08:00"
}
```

必填建议：

- `name`
- `ridingYears`
- `currentLevel`

建议后续必填：

- `coachName`
- `clubName`

### 2.2 TrainingVideo

视频上传记录。

```json
{
  "id": "video_001",
  "studentId": "student_001",
  "fileName": "training-20260601.mp4",
  "fileUrl": "https://example.com/videos/video_001.mp4",
  "durationSec": 68,
  "sizeMb": 82.5,
  "format": "mp4",
  "uploadStatus": "uploaded",
  "uploadProgress": 100,
  "uploadError": "",
  "analysisConsent": true,
  "caseConsent": false,
  "consentAt": "2026-06-01T10:04:30+08:00",
  "createdAt": "2026-06-01T10:05:00+08:00"
}
```

`uploadStatus` 枚举：

- `selected`
- `validating`
- `uploading`
- `uploaded`
- `failed`

授权字段说明：

- `analysisConsent`：必填，表示学员或监护人同意该视频用于本次 AI 训练分析、报告生成和教练复核。
- `caseConsent`：选填，表示允许后续作为匿名案例素材候选。公开发布前仍需运营二次确认，不可直接公开。
- `consentAt`：用户确认授权的时间。

### 2.3 AnalysisTask

AI 分析任务。

```json
{
  "id": "task_001",
  "studentId": "student_001",
  "videoId": "video_001",
  "status": "analyzing",
  "progressText": "AI 正在分析骑乘姿态",
  "retryCount": 0,
  "errorCode": "",
  "errorMessage": "",
  "createdAt": "2026-06-01T10:06:00+08:00",
  "updatedAt": "2026-06-01T10:07:00+08:00"
}
```

`status` 枚举：

- `queued`
- `analyzing`
- `generating_report`
- `completed`
- `failed`

## 3. 接口草案

接口命名可以根据实际后端改成云函数或 REST API。

### 3.1 获取学员资料

`GET /api/student/profile`

响应：

```json
{
  "profile": {}
}
```

### 3.2 保存学员资料

`POST /api/student/profile`

请求：

```json
{
  "name": "张同学",
  "age": 14,
  "heightCm": 160,
  "weightKg": 48,
  "ridingYears": 2,
  "currentLevel": "初级进阶",
  "coachName": "李教练",
  "clubName": "示例马术俱乐部"
}
```

响应：

```json
{
  "success": true,
  "profileId": "student_001"
}
```

### 3.3 创建视频上传凭证

`POST /api/videos/upload-token`

请求：

```json
{
  "fileName": "training.mp4",
  "sizeMb": 82.5,
  "durationSec": 68,
  "format": "mp4",
  "analysisConsent": true,
  "caseConsent": false
}
```

响应：

```json
{
  "videoId": "video_001",
  "uploadUrl": "https://example.com/upload",
  "maxSizeMb": 300,
  "expiresInSec": 900
}
```

错误码补充：

- `VIDEO_CONSENT_REQUIRED`：未确认视频用于本次训练分析，不允许创建上传凭证。

### 3.4 更新上传状态

`POST /api/videos/{videoId}/upload-status`

请求：

```json
{
  "uploadStatus": "uploaded",
  "uploadProgress": 100,
  "uploadError": ""
}
```

### 3.5 创建 AI 分析任务

`POST /api/analysis/tasks`

请求：

```json
{
  "videoId": "video_001",
  "studentId": "student_001"
}
```

响应：

```json
{
  "taskId": "task_001",
  "status": "queued"
}
```

### 3.6 查询 AI 分析任务

`GET /api/analysis/tasks/{taskId}`

响应：

```json
{
  "taskId": "task_001",
  "status": "generating_report",
  "progressText": "正在生成训练报告",
  "reportId": ""
}
```

### 3.7 重新分析

`POST /api/analysis/tasks/{taskId}/retry`

响应：

```json
{
  "taskId": "task_002",
  "status": "queued"
}
```

### 3.8 获取报告

`GET /api/reports/{reportId}`

响应：

```json
{
  "report": {}
}
```

### 3.9 获取历史趋势

`GET /api/students/{studentId}/trends?limit=5`

响应：

```json
{
  "limit": 5,
  "items": [
    {
      "reportId": "report_001",
      "trainingDate": "2026-06-01",
      "overallScore": 82,
      "postureControl": 80,
      "rhythmControl": 84,
      "stability": 78,
      "riskCount": 1
    }
  ],
  "trendSummary": "最近 5 次训练中，节奏控制持续提升，上身稳定性仍有波动。"
}
```

### 3.10 提交教练复核

`POST /api/reports/{reportId}/coach-review`

请求：

```json
{
  "status": "reviewed",
  "comment": "AI 判断基本准确，下次重点训练轻快步稳定性。",
  "trainingFocus": [
    "轻快步节奏",
    "小腿稳定"
  ]
}
```

## 4. 小程序本地状态建议

建议本地缓存：

- 当前学员资料
- 当前上传任务
- 当前分析任务 ID
- 最近一次报告摘要

不要长期缓存：

- 视频真实访问地址
- 个人敏感信息明文备份
- 完整报告历史
