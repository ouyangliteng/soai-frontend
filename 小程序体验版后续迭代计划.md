# 马术 AI 教学小程序体验版后续迭代计划

## 1. 当前结论

体验版测试成功，说明主链路已经打通：

学员端上传视频 -> AI 分析 -> 展示分析报告

下一阶段目标不是马上扩大功能，而是先把这条链路做成可稳定交付、可让教练和学员反复使用的产品闭环。

建议按三条线推进：

1. 产品完善：把当前小程序做到可用、稳定、专业。
2. 教练端：加入教练复核、学员管理、训练跟进。
3. 运营上线：准备内测、内容推广、销售与私域承接。

本阶段优先做第一条线：当前小程序可用化。

## 2. 第一阶段目标：把当前小程序做到可用

阶段目标：

- 学员可以完整填写基础资料。
- 上传视频时状态清晰，失败可重试。
- AI 报告从简单评分升级为训练建议型报告。
- 可以查看最近 5 次、10 次训练趋势。
- 教练可以对 AI 报告进行复核和补充意见。

建议版本定义：

- V0.1 体验版：已完成主链路验证。
- V0.2 内测可用版：补齐资料、报告、历史趋势、上传体验。
- V0.3 教练协作版：加入教练端学员管理和复核流程。

## 3. P0 功能清单

### 3.1 学员资料

需要补充的字段：

- 姓名
- 年龄
- 身高
- 体重
- 骑龄
- 当前等级
- 所属教练
- 所属俱乐部

建议页面：

- 学员资料页
- 首次使用资料补全弹窗或引导页
- 资料编辑页

建议交互规则：

- 首次上传视频前，引导用户补全基础资料。
- 姓名、当前等级、骑龄、所属教练建议设为重点字段。
- 年龄、身高、体重用于训练风险判断，但可以允许后续补充。
- 教练和俱乐部字段后续要能关联教练端账号。

验收标准：

- 新用户首次进入可以填写资料。
- 老用户可以编辑资料。
- 上传视频时可以读取学员资料并进入 AI 分析上下文。
- 报告页可以显示学员基础信息摘要。

### 3.2 报告内容优化

当前报告如果只是评分展示，价值还不够。建议升级为训练建议型报告。

报告模块建议包括：

- 总体评分
- 问题点
- 风险点
- 本次进步
- 下次训练重点
- 教练复核意见
- 历史趋势

报告结构建议：

```json
{
  "student": {
    "name": "学员姓名",
    "level": "当前等级",
    "ridingYears": "骑龄",
    "coach": "所属教练",
    "club": "所属俱乐部"
  },
  "summary": {
    "overallScore": 82,
    "levelAssessment": "基础稳定，转弯阶段需要提升身体控制",
    "trainingDate": "2026-06-01"
  },
  "problemPoints": [
    {
      "title": "上身前倾",
      "detail": "快步阶段身体重心略向前，影响骑坐稳定性",
      "severity": "medium"
    }
  ],
  "riskPoints": [
    {
      "title": "膝部夹持过紧",
      "detail": "可能导致小腿位置漂移，影响扶助准确性",
      "riskLevel": "medium"
    }
  ],
  "improvements": [
    "本次节奏控制比上次更稳定",
    "转弯前视线方向更明确"
  ],
  "nextTrainingFocus": [
    "保持肩、髋、脚跟垂直线",
    "练习轻快步时小腿稳定性",
    "加强转弯前的提前视线引导"
  ],
  "coachReview": {
    "status": "pending",
    "comment": ""
  },
  "trend": {
    "last5": [],
    "last10": []
  }
}
```

页面展示建议：

- 顶部：学员信息、训练时间、总评分。
- 第一屏重点：总体结论、主要问题、风险提醒。
- 第二屏：本次进步、下次训练重点。
- 第三屏：历史趋势、教练复核意见。

验收标准：

- 报告不只显示分数，必须显示可执行训练建议。
- 问题点和风险点要分开，避免把技术问题和安全风险混在一起。
- 教练复核意见可以为空，但页面要预留状态。
- 报告语言要专业、准确、克制，避免夸张或不确定判断。

### 3.3 历史趋势

目标：

让学员和教练看到最近训练是否进步，而不是只看单次报告。

建议展示：

- 最近 5 次训练变化
- 最近 10 次训练变化
- 总评分趋势
- 稳定性趋势
- 姿态控制趋势
- 节奏控制趋势
- 风险次数变化

建议图表：

- 折线图：总评分、姿态、节奏、稳定性。
- 条形或标签统计：风险点出现次数。
- 简短趋势结论：例如“最近 5 次训练中，节奏控制持续提升，但上身稳定性波动较大。”

验收标准：

- 学员至少能看到最近 5 次训练记录。
- 有 10 次以上训练时展示 10 次趋势。
- 少于 5 次时显示空状态和提示。
- 趋势结论必须基于历史数据生成，不能只写固定文案。

### 3.4 上传体验

需要补齐：

- 视频上传进度
- 分析中状态
- 失败重试
- 视频过大提示
- 网络异常提示
- 上传格式提示

建议状态流：

1. 待选择视频
2. 视频校验中
3. 上传中
4. 上传成功
5. AI 分析中
6. 分析完成
7. 分析失败，可重试

建议提示：

- 视频过大：提示压缩或重新选择较短视频。
- 网络失败：保留当前视频选择，允许重新上传。
- 分析失败：允许重新发起分析，并提示联系教练或客服。
- 分析中：显示预计等待，不让用户误以为卡死。

验收标准：

- 上传过程中必须有百分比或明确进度状态。
- 上传失败后用户不需要重新填写资料。
- 分析中页面刷新后仍能恢复状态。
- 视频过大时在上传前拦截，而不是上传后失败。

## 4. 推荐开发顺序

### 第 1 周：资料和上传稳定性

1. 增加学员资料数据结构。
2. 增加资料填写和编辑页面。
3. 上传前检查资料是否完整。
4. 增加视频大小、格式、时长校验。
5. 增加上传进度、失败重试、分析中状态。

### 第 2 周：报告升级

1. 调整 AI 报告输出结构。
2. 报告页加入问题点、风险点、进步、下次训练重点。
3. 预留教练复核意见模块。
4. 增加历史训练记录入口。
5. 展示最近 5 次趋势。

### 第 3 周：趋势和内测准备

1. 增加最近 10 次趋势。
2. 增加趋势结论。
3. 完成异常状态和空状态。
4. 准备 10-20 名学员内测。
5. 收集报告准确性、可读性、上传成功率反馈。

## 5. 数据指标

上线内测前建议重点看：

- 视频上传成功率
- AI 分析成功率
- 平均分析等待时间
- 报告页打开率
- 报告完整阅读率
- 教练复核率
- 学员二次上传率
- 最近 7 天留存

建议内测目标：

- 上传成功率 >= 90%
- AI 分析成功率 >= 85%
- 报告页打开率 >= 95%
- 学员二次上传率 >= 30%
- 教练认为报告有参考价值的比例 >= 70%

## 6. 产品语气与视觉方向

产品表达要符合公司定位：

- 实用
- 专业
- 技术真实
- 高级科技感
- 不夸张承诺
- 不把 AI 包装成替代教练，而是定位为教练辅助工具

建议核心文案：

- AI 训练分析报告
- 动作问题识别
- 骑乘风险提醒
- 训练进步追踪
- 教练复核建议

避免文案：

- AI 完全替代教练
- 绝对准确
- 一次训练快速提升
- 自动判断所有风险
